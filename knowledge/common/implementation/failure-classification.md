# 实现失败分类

## 目的

把失败导向正确责任域，避免所有错误都变成目标代码修改。

## 分类

### 模型事实缺失

表现：无法确定模型真实语义、权重布局或参考行为。

动作：返回 `model-analyze --update`。

### 评估缺口

表现：发现新的 capability、目标架构判断错误或工作项粒度不合理。

动作：重新 `adapt-assess`。

### 环境问题

表现：包来源、版本、设备、动态库、资源或启动环境与评估不一致。

动作：停止代码修改，修复环境后重评估。

### 实现缺陷

表现：工作项和路径明确，局部实现未满足预期。

动作：提出一个新的、可证伪的局部假设。

### 测试 oracle 缺失

表现：命令能运行，但没有可靠标准判断结果正确。

动作：补充局部测试或请求开发者决定，不以“无报错”代替正确性。

### 范围或依赖边界

表现：需要升级依赖、修改上游核心、改变公共接口或处理另一个 capability。

动作：生成开发者决策或返回评估拆项。

### 瞬态执行失败

表现：设备占用、调度中断、网络或临时进程问题，且有直接证据。

动作：允许一次受控重试。第二次仍失败则重新分类。
