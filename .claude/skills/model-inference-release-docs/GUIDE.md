# model-inference-release-docs 指南

## 输入材料

输入不要求是完整表格，可以是：

```text
CSV / TSV / Excel
Markdown 表格
测试结果截图转写
benchmark 日志摘录
用户口述的模型、镜像、性能、精度和部署信息
```

优先识别以下信息：

```text
模型、框架、版本、日期、镜像、权重地址
精度数据集和分数
在线测试、吞吐测试、长文测试、泛化测试结果
环境变量、容器命令、服务启动命令、请求示例
```

## 生成方式

由模型读取输入材料，按 `references/template.md` 生成完整 Markdown。不要调用固定 Python 转换脚本。

## 文档模板

生成文档固定包含：

```text
1. 概览
2. 精度测试
3. 性能测试
   3.1 在线测试（1K & 2K）
   3.2 吞吐场景测试
   3.3 长文测试（必选）
   3.4 泛化测试（必选）
4. 功能（可选）
5. 推理指南
```

## 规则

- 表格或材料中已有的测试数据必须原样保留；
- 缺失的部署步骤、命令结构、章节说明由模型按模板补全；
- 无证据的数据项写 `未提供`，不要编造数值；
- 对用户未给出的镜像名、模型路径、端口、TP 等参数可使用尖括号占位；
- 不读取本仓模型适配阶段文件。
